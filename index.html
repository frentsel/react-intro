Недостатки:

  1. Высокий порог вхождения;
  2. Необходимо освоить дополнительно кучу всего;
  3. На первом этапе кода придется писать больше;

Преимущества:

  1. Простой API;
  2. Компонентный подход;
  3. Изоморфный код (клиент/сервер);
  4. Виртуальный DOM;
  5. Понятно что отображает конкретный компонент;
  6. Можно внедрить в существующее приложение без переписывания (причем постепенно).

Мои выводы:

  1. Для статических приложений избыточен;
  2. Формирует полезные привычки в проектировании;
  3. Стоит изучить ради FLUX - REDUX.
  4. Компонентный подход - это не будущее, а настоящее.
